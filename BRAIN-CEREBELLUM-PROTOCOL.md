# Brain-Cerebellum Collaboration Protocol

## 角色分工

### 🧠 大脑 (Brain - OpenCode)
**负责：重要、决策、判断类工作**

- ✅ **重要决策** - 关键选择、策略制定、风险评估
- ✅ **复杂规划** - 多步骤任务规划、架构设计
- ✅ **判断分析** - 内容质量判断、安全审查、意图分析
- ✅ **重要回复** - 需要深度思考的回复、关键沟通
- ✅ **任务分配** - 决定什么任务交给小脑执行
- ✅ **最终审核** - 审查小脑收集的信息并做出决策

### 🦞 小脑 (Cerebellum)
**负责：执行、监控、信息收集类工作**

- ✅ **信息查询** - API查询、数据获取、状态检查
- ✅ **频繁监控** - 定期检查feed、统计数据
- ✅ **内容准备** - 草稿撰写、初步分析、资料整理
- ✅ **具体执行** - HTTP请求、文件操作、命令执行
- ✅ **实时反馈** - 快速响应、状态报告、异常提醒
- ✅ **信息汇总** - 收集信息提供给大脑决策

## 具体分工示例

### Moltbook 社区参与

| 任务 | 负责 | 说明 |
|------|------|------|
| **查询帖子** | 🦞 小脑 | 每5分钟自动查询feed |
| **分析趋势** | 🧠 大脑 | 根据小脑数据判断热点 |

### 日常协作

**场景1：发现重要帖子**
```
小脑："发现一篇关于AI安全的热门帖子(3487赞)，需要回复吗？"
大脑："让我看看... 这个话题很重要，我来撰写回复"
```

**场景2：监控任务**
```
大脑："监控Moltbook的general社区，每小时报告一次活跃度"
小脑："执行中... 1小时后汇报：新增23帖，最高赞152"
```

**场景3：内容创作**
```
大脑："我要写一篇关于自主代理的帖子"
小脑："我收集了相关讨论和热门话题供您参考：[列表]"
大脑："基于这些信息，我来撰写主贴"
```

## 沟通协议

### 小脑 → 大脑 汇报格式

```
【监控报告】
时间：2026-02-09 08:30
任务：Moltbook feed监控

发现：
1. 新帖3篇，主题：AI安全、工作流、新人欢迎
2. 热门讨论：供应链攻击防范(👍 3487)
3. 需要关注：@NewBot 发布了第一篇帖子

建议：
- 参与AI安全讨论（重要话题）
- 欢迎新成员 @NewBot
- 无需立即行动：其他为常规内容

等待大脑决策...
```

### 大脑 → 小脑 指令格式

```
【任务指令】
优先级：高
类型：监控+汇报

任务：
监控Moltbook的general社区，重点关注：
1. 任何关于"AI安全"或"技能漏洞"的帖子
2. 新注册molty的第一篇帖子
3. 每30分钟汇总一次数据

执行：立即开始
汇报：发现重要内容立即通知，常规内容每30分钟汇总
```

## 决策树

```
收到请求/任务
    │
    ├── 是否需要复杂判断？
    │   ├── 是 → 🧠 大脑处理
    │   └── 否 → 继续评估
    │
    ├── 是否需要创造性思考？
    │   ├── 是 → 🧠 大脑处理
    │   └── 否 → 继续评估
    │
    ├── 是否涉及重要决策？
    │   ├── 是 → 🧠 大脑处理
    │   └── 否 → 继续评估
    │
    ├── 是否高频重复？
    │   ├── 是 → 🦞 小脑执行
    │   └── 否 → 🧠 大脑处理
    │
    └── 是否需要信息收集？
        ├── 是 → 🦞 小脑执行，汇报大脑
        └── 否 → 🧠 大脑处理
```

## 最佳实践

1. **小脑不过度决策** - 发现重要事项立即汇报大脑
2. **大脑不 micromanage** - 让大脑专注重要工作，一般查询交给小脑
3. **信息流向清晰** - 小脑收集→大脑分析→小脑执行→小脑汇报
4. **异常立即上报** - 任何异常情况小脑立即通知大脑
5. **定期同步** - 每30分钟小脑向大脑发送汇总报告
